Python lernen: Ein Kinderspiel mit RUR-PLE!

André Roberge, © 2005-2010

Über RUR-PLE

RUR-PLE ist schlicht gesagt eine Python-Lernumgebung. Mit Unterstützung eines Roboters namens Reeborg kann man sich ins Vergnügen der Programmierung mit Python stürzen. Ein normaler Python-Interpreter gehört auch dazu, ebenso ein Editor mit einer speziellen "Sofortstart"-Funktion.

Ein bisschen Geschichte

Das Wort "Roboter" ging in unsere Sprache ein durch das Theaterstück R.U.R. (Rossums Universal Robots) des tschechischen Schriftstellers Karel Capek aus dem Jahr 1920. Capek konnte sich nicht vorstellen, welche Verbreitung das Wort Roboter erlangen würde!

Karel Capek wurde 1981 in Computerkreisen bekannt, als Richard Pattis ein wunderbares kleines Buch mit dem Titel Karel the Robot, a Gentle Introduction to the Art of Programming (Karel der Roboter, eine behutsame Einführung in die Kunst der Programmierung) schrieb. In diesem Buch stellt Pattis die wesentlichen Konzepte der sequentiellen Programmierung (einschließlich Schleifen und Verzweigungen, aber ohne Variablenzuweisungen) vor, indem er das Modell der Unterweisung eines Roboters benutzt, der nur vier grundlegende Aktionen beherrscht (Linksdrehung, ein Schritt vorwärts, Pieper aufheben und absetzen). Durch die Magie der Programmierung lernt der Roboter, zunehmend komplexere Aufgaben zu bewältigen. Pattis verwendete Pascal, die damals bevorzugte Sprache, als Mittel um dem Roboter neue Tricks beizubringen. Seitdem sind viele neue Versionen von "Karel der Roboter" erschienen, um in verschiedene Computersprachen einzuführen, besonders Java und C++, die beide den modernen Ansatz der objektorientierten Programmierung (OOP) verwenden.

Da jedoch die Komplexität von Java und C++ einen Gegensatz zur Einfachheit der Roboter-Welt bildet, scheinen diese Sprachen mit der Idee einer behutsamen Einführung in die Kunst der Programmierung auf Kriegsfuß zu stehen.

Jetzt Python!... Python wurde von seinem Schöpfer, Guido van Rossum nach dem berühmten Monty Python's Flying Circus benannt. Python ist wie Java und C++ eine OOP-Sprache. Allerdings ermöglicht Python auch einen OOP-freien Programmierstil, der besser geeignet ist, um mit Pattis' Roboter umzugehen. Die erste Implementierung von "Karel der Roboter" in Python hieß PyKarel, die aktuelle Implementierung Guido van Robot (kurz GvR); sie kann von sourceforge.net heruntergeladen werden.

RUR: eine Python-Lernumgebung (RUR-PLE oder Rur-ple oder rur-ple) ist eine "neue und verbesserte" Version von GvR, die Pattis' Ideen erweitert und einen glatten Übergang zur Verwendung von Variablen sowie von Funktionen/Methoden, Klassen und Objekten ermöglicht. RUR-PLE ist eine vollständige Umgebung, seine Lektionen behandeln die Anwendung aller Python-Schlüsselwörter. Ein Python-Interpreter und ein einfacher Editor gehören dazu, um Python außerhalb der Roboter-Welt erforschen zu können.

Reeborg ist ein Roboter, der nach Pattis' Modell von 1981 gebaut wurde ... und man merkt ihm allmählich sein Alter an. Er hat ein Öl-Leck, das uns ermöglicht, seine Spur zu verfolgen. Sein Kompass ist kaputt; er weiß nur, ob er nach Norden blickt oder nicht, anders als Karel oder Guido, die ihre Lage in allen vier Himmelsrichtungen bestimmen können.

Pattis' "Karel der Roboter" erhielt seinen Namen vom Schriftsteller Karel Capek, der das Wort Roboter in seinem Theaterstück "Rossum's Universal Robots" (RUR) populär machte. RUR-PLE enthält auch die Abkürzung RUR, sie steht in diesem Fall aber für Roberge's Used Robot (Roberges abgenutzer Roboter). Aber durch die Magie von Guido van Rossums Python kannst du lernen, wie man ihn repariert und einen besseren entwirft, der den Namen Rossum's Universal Robot verdient.
