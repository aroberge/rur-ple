27. Variablen

Erinnerst du dich an die Übersetzungen, die wir im 5. Kapitel einführten? Wir sagten, dass du Reeborg andere Sprachen beibringen kannst. Als Beispiel benutzten wir das deutsche Wort für "move".

geh = move

Wir hatten move einfach ein deutsches Wort als Name zugewiesen. Auch Zahlen können wir Namen zuweisen.

>>> n = 3
>>> print n
3
>>> 2*n
6
>>> n*n
9

Feine Sache, nicht wahr? Jetzt können wir den Wert des Namens nach Belieben ändern.

>>> n = 3
>>> print n
3
>>> n = 2
>>> print n
2

Weil der Wert des Namens sich ändern kann, bezeichnen wir ihn als Variable. Wenn du einen Namen vergibst, sagen wir, dass du einer Variablen einen Wert zuweist. Das Zeichen "=" bedeutet "ist ein Name für" und nicht "ist gleich", wie du es wohl gewohnt bist. Denn wenn "n gleich 2" ist, dann ist sicher auch "2 = n". Lass uns mal schauen, was Python daraus macht.

>>> n = 2    # Das ist ok.
>>> 2 = n
SyntaxError: can't assign to literal

Wenn es auf Standardtastaturen eine Alternative zum Gleichheitszeichen gegeben hätte wie zum Beispiel

>>> n ← 3
>>> print n
3

hätte Guido van Rossum, der Autor von Python, es sicher als Symbol für "ist ein Name für" genommen. Das gibt es aber nicht und deshalb müssen wir statt dessen das Gleichheitszeichen benutzen. Das erklärt auch, warum wir ein doppeltes Gleichheitszeichen nehmen müssen, wenn wir zwei Zahlen auf Gleichheit prüfen wollen.

Wie wir oben kurz gesehen haben, kann eine Variable nach Zuweisung einer Zahl genau wie diese Zahl eingesetzt werden. Zum Beispiel:

>>> a = 1
>>> b = 2
>>> c = a + b
>>> print c
3

Was wir gerade gesehen haben, sollte nicht allzu schwer zu verstehen sein. Dies hier dürfte auf den ersten Blick ein bisschen schwieriger sein:

>>> a = 1
>>> a = a + a   # Ueberraschung?
>>> print a
2

Denk daran, dass "=" "ist ein Name für" bedeutet. Daher ist die "überraschende" Zeile so zu verstehen:

  • Nimm, was auf der rechten Seite des Symbols "=" steht (a + a.)
  • Berechne seinen Wert (2; Ergebnis von 1 + 1).
  • Für dieses Ergebnis wird die Variable auf der linken Seite (a) als Name vergeben.

Wenn ich das alles ausschreibe, klingt es sehr kompliziert. Aber tatsächlich wirst du dich schnell daran gewöhnen, und es wird sehr bald keine Überraschung mehr für dich sein.

Weil diese Methode der Wertänderung einer Variablen so häufig vorkommt, gibt es in Python eine Kurzform dafür:

>>> a = 1
>>> a += 3       # entspricht "a = a + 3"
>>> print a
4

>>> a -= 2       # entspricht "a = a - 2"
>>> print a
2

>>> a *= 5       # entspricht "a = a * 5"
>>> print a
10

>>> a /= 2       # entspricht "a = a / 2"
>>> print a
5

Du bist dran.

Erinnerst du dich, wie schwierig es für Reeborg war, zwei Dezimalzahlen zu addieren? Um dein Gedächtnis ein wenig aufzufrischen, soll Reeborg zwei Zahlen auf folgende Weise addieren:

3+2=5

8+4=12

Du kannst Variablen verwenden, um dir die Aufgabe zu erleichtern; wenn du das tust, brauchst du keine besondere Welt wir beim letzten Mal und Reeborg muss auch keine Piepser bei sich tragen um anzufangen. Hier ein paar Vorschläge:

  • Verwende ein Variable mit dem Namen summe, deren Wert anfangs 0 ist.
  • Lass Reeborg die Piepser einzeln aufheben, wobei der Wert von summe jedes Mal um 1 erhöht wird, zum Beispiel mit der Kurzform
    sum += 1.
  • Wenn Reeborg fertig ist, betrachte zwei Fälle: summe ist größer als 10 oder nicht. Wenn ja, lass Reeborg alle Piepser bis auf 10 ablegen, einen Schritt zur Seite gehen und einen ablegen (den Übertrag). Wenn nein, lass Reeborg alle Piepser ablegen.
  • Dringende Empfehlung: Definiere keine neue Funktion außer turn_right(). Den Grund wirst du einige Lektionen später erfahren.
